Before you begin, have these ready: Your @coursemojo.com Google account, and your Anthropic account sign-in (the same one you use for Claude).

Step-by-step

1

Install Google Drive for Desktop

Google Drive for Desktop syncs the shared drive to your computer so Claude can access curriculum files directly.

  1. Go to google.com/drive/download
  2. Click "Download Drive for desktop"
  3. Open the downloaded file and follow the installer prompts
  4. Once installed, sign in with your @coursemojo.com Google account
  5. Verify it's working: open Finder (Mac) or File Explorer (Windows) and look for "Google Drive" in the sidebar. You should see Shared drives › 2026-27 Activity Creation
Don't see the shared drive? Make sure you're signed into the correct Google account. Click the Drive icon in your menu bar / system tray › Settings › confirm your coursemojo account is listed.
2

Install VS Code

VS Code is a free code editor from Microsoft. You'll use it as the place where you talk to Claude.

  1. Go to code.visualstudio.com/download
  2. Download the version for your computer:
 MacClick the big blue Mac button (it auto-detects Intel vs. Apple Silicon). Then drag the VS Code icon into your Applications folder and open it from there.
⚹ WindowsClick the big blue Windows button, run the installer, and follow the prompts. Check "Add to PATH" if asked.
  1. Open VS Code. You'll see a Welcome tab — you can close it.
3

Install the Claude Code Extension

This connects Claude to VS Code so you can chat with it directly.

  1. Open VS Code
  2. Click the Extensions icon in the left sidebar (it looks like four small squares)
  3. In the search bar at the top, type: Claude Code
  4. Find "Claude Code" by Anthropic and click Install
  5. After install, you'll see a Claude icon in your left sidebar (it looks like a sparkle / star ✨)
4

Sign Into Claude

  1. Click the Claude icon in the VS Code sidebar
  2. Claude will prompt you to sign in — click the sign-in link
  3. Your browser opens. Sign in with your Anthropic account (the one you already use for Claude)
  4. Return to VS Code — Claude should now be active and ready
5

Open the Shared Drive in VS Code

This tells Claude where to find the curriculum files.

  1. In VS Code, go to File › Open Folder…
  2. Navigate to Google Drive › Shared drives › 2026-27 Activity Creation
  3. Click Open
  4. If VS Code asks "Do you trust the authors of the files in this folder?" — click Yes, I trust the authors
Almost there! Claude can now see all the shared drive files. One more step — connecting Jira so Claude can move your tickets.
6

Connect Jira

This lets Claude update Jira tickets on your behalf — moving them along the board when you run /move-to-review, /move-to-final, or /revisions.

6.1  Create your Jira API token

  1. Go to id.atlassian.com › API tokens
  2. Click the blue "Create API token" button
  3. Give it a name — something like Claude Code — and click Create
  4. A long string of characters appears. Click Copy
  5. Keep this browser tab open until you finish 6.2 — you won't be able to see the token again once you close it

6.2  Save the token to a config folder

 Mac 1. Finder › Desktop › right-click › New Folder › name it config
2. Open TextEdit, then Format › Make Plain Text (Shift + Cmd + T) — important, it must be plain text
3. Add the two lines below, then File › Save as jira-token.txt inside the config folder
⚹ Windows 1. Right-click Desktop › New › Folder › name it config
2. Inside it, right-click › New › Text Document › name it jira-token.txt
3. Open it in Notepad, add the two lines below, then Ctrl + S and close
email: your.name@coursemojo.com token: <paste the token you copied in 6.1 here>

6.3  Let Claude finish the setup

  1. Open Claude Code in VS Code (click the Claude icon in the sidebar)
  2. Copy the entire prompt below, paste it into the Claude chat, and press Enter:
Please set up my Jira credentials for the move-to-review skill.

Read the file at ~/Desktop/config/jira-token.txt — it contains two lines:
  email: <my email>
  token: <my Jira API token>

Parse those values, then create (or overwrite) the file at ~/.jira-config with exactly this format, substituting my email and token:

JIRA_DOMAIN=coursemojo.atlassian.net
JIRA_EMAIL=<my email>
JIRA_API_TOKEN=<my token>

Then verify the credentials work by running:
  curl -s -o /dev/null -w "%{http_code}" -u "<email>:<token>" https://coursemojo.atlassian.net/rest/api/3/myself

If it returns 200, reply "Jira setup complete — you can now use /move-to-review." If it returns anything else, tell me the response code and what might be wrong so I can fix the token file.
  1. Claude reads your token file, configures everything, and tests the connection. You should see a success message within a few seconds.
🔒 Your token is sensitive. Treat it like a password — don't share your config folder or the file inside. If you ever suspect someone else has seen it, return to the Atlassian page in 6.1 and revoke/recreate the token.
Nearly there! Claude can now read the shared drive and update your Jira tickets. One quick install to go.
7

Install Ask Mojo (your help desk)

Ask Mojo is a friendly panel inside VS Code — ask it anything about the pipeline, the process, or where to find things, and it answers from the team's guide.

  1. In VS Code, open the Extensions panel (the four-squares icon, or Cmd/Ctrl + Shift + X)
  2. Click the ⋯ ("More Actions") menu at the top of that panel, then choose Install from VSIX…
  3. Navigate to Google Drive › Shared drives › 2026-27 Activity Creation › ask-mojo-extension and pick ask-mojo-latest.vsix
  4. When VS Code offers to Reload, click it
  5. A new Ask Mojo icon appears in the left Activity Bar — click it and try a question like "Where do I find my module plan?"
You're all set! 🎉 You're ready to author — and Mojo's right there whenever you get stuck.

Opening a different folder later

You can change which folder Claude is looking at any time — for example, switching to a specific curriculum or a draft-output folder. The folder you open is the world Claude can see.

  1. Go to File › Open Folder… (shortcut: Cmd + O on Mac, Ctrl + O on Windows)
  2. Navigate to the folder you want and click Open
  3. VS Code reloads with that folder as the active workspace — Claude can now see everything inside it
  4. Confirm which folder is open in the Explorer panel (top icon in the left sidebar) — it shows the folder name and its contents
Tip: To get back to the main shared drive, do File › Open Folder… again and navigate back to Google Drive › Shared drives › 2026-27 Activity Creation.
