For This to Work, Two Things Must Be Right

1. Module Plan

Claude reads your module plan as its blueprint. The Activity ID is how it knows what to build.

No Activity ID = nothing happens.

2. Source Materials

Claude looks for texts, lesson plans, and teacher materials in the Curriculum Hub. If the folder is empty, the build fails.

Missing materials = failed build.

These are the #1 and #2 reasons builds fail. Your prep work is what makes Claude's drafts good.

The Module Plan Is Claude's Blueprint

Everything Claude builds comes from what you put in the module plan.

What Claude Reads

  • Activity ID — what to build (must match Jira ticket name)
  • Question counts — how many of each type
  • Question types — LCQ, TDQ, VOC, etc.
  • Excerpt references — which text to use
  • Verbatim content — exact wording requirements

What Makes a Great Plan

  • Activity ID present and correct
  • Specific question types (not just "3 questions")
  • Clear excerpt references (paragraph numbers)
  • Any verbatim text spelled out exactly
  • The more specific you are, the better the draft

What You Actually Do

  1. Finalize your module plan — make sure Activity IDs are present and specs are detailed
  2. Confirm source materials are in place — check with Steven if unsure
  3. Move your ticket to "Draft Activity with Claude" on the Jira board
  4. Wait ~15-20 minutes — the ticket moves to Draft Review when the draft is ready
  5. Open the draft in Claude Code and edit it — ask Claude questions, make changes
  6. Run /move-to-review when you're done — picks the reviewer, notifies them automatically
You never manually drag tickets past "Draft Activity with Claude" — the skills handle everything.

How You Know What's Happening

The board and Jira notifications keep you in the loop automatically.

Draft is ready

Your ticket moves to Draft Review. Open draft_outputs/ on the shared drive to find it.

Lead sends it back

You get a Jira notification with specific revision notes. Ticket moves back to Draft Review.

Build failed

Ticket stays in Draft Activity with Claude. A Jira comment explains what went wrong (usually missing source materials).

Activity finalized

Transformation team gets notified automatically. You're done with that activity.

Three Skills You'll Use

Type these commands in Claude Code. They handle Jira + files + notifications for you.

/move-to-review

Done editing? This asks you who should review it (Meg or Eric), moves the ticket, and notifies them.

/revisions

Lead sends it back? They run this to tell you what needs fixing. You get a notification with their notes.

/move-to-final

Lead approves? This finalizes the activity and notifies the transformation team. You're done.
